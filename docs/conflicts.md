## [CART-01] `DELETE /cart/selected` 제거

- **발생일**: 2026-04-24
- **관련 명세**: CartServer API 명세서 (2026-04-24 업데이트)
- **상태**: ✅ 해결 — 방안 A 선택 (선택삭제 기능 제거, 개별 삭제만 제공)

### 내용

기존 명세에 있던 `DELETE /api/v1/cart/selected` (선택 항목 일괄 삭제) 엔드포인트가 새 명세에서 제거됨.

### 영향 범위

| 파일 | 변경 내용 |
| ---- | --------- |
| `src/api/cartApi.js` | `removeSelectedCartItems` mutation 제거 |
| `src/pages/CartPage.jsx` | 「선택삭제」 버튼 및 `handleRemoveSelected` 핸들러 제거 |

### 현재 상태

선택삭제 기능이 UI에서 완전히 제거된 상태. 사용자는 항목별 「삭제」 버튼으로만 개별 삭제 가능.

### 대체 방안 후보

| 방안 | 설명 | 장단점 |
| ---- | ---- | ------ |
| **A. 현행 유지** | 선택삭제 기능 없이 개별 삭제만 제공 | 구현 비용 없음 / UX 불편 |
| **B. 프론트 순차 호출** | 체크된 항목에 대해 `DELETE /cart`를 순차 호출 | UX 유지 / N번 API 호출 |
| **C. 백엔드 재추가 요청** | 서버팀에 `DELETE /cart/selected` 재추가 요청 | 깔끔 / 백엔드 일정 필요 |

### 결정 내용

방안 A 채택 (2026-04-24). 선택삭제 기능 제거, 항목별 「삭제」 버튼으로 개별 삭제만 제공.
